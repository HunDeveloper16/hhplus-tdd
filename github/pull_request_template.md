### 기술적 성장
-새로 학습한 개념

ReentrantLock의 장점:
특정 임계 구간만 따로 명시적으로 락을 획득하고 해제할 수 있다.
Synchronized는 공정성 옵션이 따로없지만 ReentrantLock은 락 요청 순서대로 접근할 수 있는 공정성 옵션을 사용할 수 있다.
이에따라, 기아(Starvation)를 방지할 수 있다.

TDD의 전반적인 작성:
전반적으로 구간을 나눠 테스트 코드를 작성하는 방법을 알게된 것 같다.
